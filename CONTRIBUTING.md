# 贡献指南

感谢您有兴趣为《从零到一：羽毛球系统教学与训练体系》项目做出贡献！本指南将帮助您了解如何参与贡献。

## 贡献方式

### 1. 报告问题
如果您发现任何问题，包括：
- 技术错误
- 内容错误
- 排版问题
- 链接失效
- 其他任何问题

请通过GitHub Issues提交问题报告。

### 2. 提出建议
如果您有改进建议，包括：
- 新的训练方法
- 更好的讲解方式
- 额外的训练内容
- 用户体验改进
- 其他任何建议

请通过GitHub Issues提交建议。

### 3. 提交代码
如果您想直接贡献代码或内容：
1. Fork本仓库
2. 创建特性分支 (`git checkout -b feature/AmazingFeature`)
3. 提交更改 (`git commit -m 'Add some AmazingFeature'`)
4. 推送到分支 (`git push origin feature/AmazingFeature`)
5. 打开Pull Request

## 贡献内容类型

### 文档改进
- 修正错别字和语法错误
- 改进文档结构和可读性
- 补充缺失的内容
- 优化示例和说明

### 技术内容
- 补充训练方法和技术细节
- 添加科学依据和参考文献
- 完善评估工具和标准
- 优化训练计划和周期安排

### 翻译工作
- 翻译为其他语言
- 完善现有翻译
- 添加多语言支持

### 工具和资源
- 创建训练工具
- 添加训练视频链接
- 制作训练图表和示意图
- 开发辅助应用程序

## 贡献规范

### 文档规范
1. **风格统一**：保持口语化讲解 + 专业表格的混合风格
2. **格式规范**：使用Markdown格式，保持一致的标题层级
3. **语言规范**：使用规范的中文，避免网络用语和口语化过度
4. **专业准确**：确保技术内容的科学性和准确性

### 代码规范
1. **代码风格**：遵循项目现有的代码风格
2. **注释规范**：添加必要的注释说明
3. **测试完善**：确保代码的正确性和稳定性
4. **文档更新**：更新相关的文档说明

### 提交规范
1. **提交信息**：使用清晰、简洁的提交信息
2. **原子提交**：每次提交只解决一个问题
3. **分支管理**：合理使用分支，保持主分支稳定
4. **PR描述**：在Pull Request中详细描述更改内容

## 开发环境设置

### 本地开发
1. 克隆仓库：`git clone https://github.com/your-username/badminton-training-system.git`
2. 安装依赖：本项目主要使用Markdown文档，无需特殊依赖
3. 本地预览：使用Markdown编辑器或预览工具查看文档

### 文档编辑工具推荐
- Visual Studio Code + Markdown插件
- Typora
- Obsidian
- 其他Markdown编辑器

## 内容审核流程

### 审核标准
1. **准确性**：内容必须科学准确
2. **实用性**：内容必须对用户有用
3. **一致性**：内容必须与项目风格一致
4. **完整性**：内容必须完整，没有明显缺失

### 审核流程
1. 提交Pull Request
2. 自动检查（如果有配置）
3. 维护者审核
4. 反馈和修改
5. 合并到主分支

- 使用建设性和尊重的语言

- 保持专业和礼貌

### 合作精神
- 愿意帮助他人
- 接受他人的帮助
- 共同推动项目发展
